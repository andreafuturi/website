# AGENTS.md

## Cursor Cloud specific instructions

### Project overview
Andrea Futuri's portfolio site — a Deno SSR + Preact framework ("Singularity") with Vite for client-side dev tooling. See `README.md` and `docs/` for framework details.

### Markup — stay minimal

Do **not** add wrapper `div`s (or extra nodes) unless there is no reasonable way to achieve the layout or behavior with the existing tree — prefer grid/flex on a parent, `display: contents` where appropriate, and sizing on the real element (e.g. SVG + `object-fit`). Extra markup should be the exception and called out in review.

| Action | Command | Notes |

### Non-obvious caveats

- **Deno must be installed separately** — it is not an npm dependency. Install via `curl -fsSL https://deno.land/install.sh | sh` and ensure `~/.deno/bin` is on PATH.
- **No lockfile committed** — `npm install` generates `package-lock.json` locally but it is not tracked. `npm install` is idempotent and safe to re-run.
- **`--unstable` flag warning** — Deno 2.x prints a warning about `--unstable` being removed; this is cosmetic and does not affect operation.
- **Tests require running servers** — SSR and routing tests hit `localhost:8000`, so the dev servers must be running via `npm start` before running `npm run test`.
- **Build output** — `npm run build` outputs to `client/dist/`. This directory is committed but regenerated by build.
- **Deno dependencies are URL imports** — pulled from `esm.sh`, `deno.land/std`, and `cdn.skypack.dev` at runtime, not from npm. See `server/importMap.json`. The `copilot-setup-steps.yml` pre-fetches all of these into `~/.cache/deno` before the agent starts, so the agent never needs outbound access to those domains. (The firewall allowlist UI in Settings → Copilot → Coding agent is only available on GitHub Enterprise Cloud; it is not needed for personal accounts using the pre-caching approach.)
- **Codex environment** — If running inside OpenAI Codex (or any sandbox where `esm.sh`/`deno.land` are unreachable), run `.codex/setup.sh` once before starting the agent. It installs Deno, runs `npm install`, and pre-fetches all remote Deno URL imports into `~/.cache/deno` so no outbound HTTPS is needed during the session. The SSL `UnknownIssuer` error that Deno raises in these environments is caused by the sandbox's TLS-intercepting proxy; pre-caching eliminates the need for those connections entirely.
- **Lint excludes `no-import-prefix`** — URL imports are by-design in this Deno project; the rule is disabled in `server/deno.json`.
